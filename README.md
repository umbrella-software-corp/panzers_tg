# Panzer TG

Онлайн PvP игра 5v5 для Telegram Mini App. Главная механика — **система наведения через движущуюся линию сведения**.

## Статус

Прототип ядра геймплея. Цель текущего этапа — проверить главный вопрос: **«механика сведения цепляет?»**. Сеть, прокачка, карты — позже.

## Стек

- **client/** — Vue 3 (HUD/UX) + PixiJS (рендер игры) + Vite
- **server/** — Node.js + ws (authoritative realtime, подключаем на этапе сети)

## Запуск

```bash
pnpm install
pnpm --filter client dev      # прототип механики сведения
pnpm --filter server dev      # WS-сервер (стаб, для будущего)
```

Открой ссылку из вывода Vite на телефоне (или в DevTools в режиме мобильного), чтобы пощупать тач-управление.

## Прототип: как играть

1. Левый **джойстик** — танковое управление: вверх — газ вперёд, влево/вправо —
   поворот корпуса (можно крутиться на месте). Танк всегда стволом вверх,
   при повороте **мир вращается вокруг него** — сектор стабилен на экране.
2. Наводишься **манёвром** — держишь врага в **секторе стрельбы** (он привязан
   к направлению корпуса; отдельного прицела нет, как в ГДД).
3. Внутри сектора ходит **линия сведения**. Жми **ОГОНЬ**, когда линия на враге
   (враг в секторе И в дальности).
4. Попал — урон по врагу. Промах — ждёшь перезарядки и следующего захода.

На карте **3 бота** с ИИ (ездят, держат дистанцию, прячутся за препятствия,
стреляют в ответ), препятствия (камень/лес), туман войны, звук боя. У игрока
HP; подбили — респаун.

Переключай класс танка (Лёгкий / Средний / Тяжёлый) — меняются ширина сектора,
скорость линии, точность и перезарядка. Это прямая проверка дизайн-гипотезы из ГДД.
